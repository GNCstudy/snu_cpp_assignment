* Process

 Process는 프로그램이 설계된대로 수행하도록 해주는 프로그램의 실행 단위이다. 
운영체제는 CPU에 실행된 Process들을 사용자가 만들고, 조정하고, 종료하도록 도와준다.
Process 동작들은 PCB(Process Control Block)의 도움과 함께 쉽게 조절이 된다.
PCB는 프로세스와 관련된 모든 중요한 정보들(ID, 자료, 상태, CPU내용 등)을 포함하는 Process의 메인이다.

* Thread

 Thread는 Process의 부분이 되는 실행 단위이다.
Process는 동시에 실행될 수 있는 여러 Thread들을 가질 수 있다.
Thread는 동시에 진행되는 프로그래밍 안에서의 실행 단위이고 Scheduler에 의해 독립적으로 관리될 수 있다.
병렬을 이용한 어플리케이션의 수행 시에 사용자에게 도움을 줄 수 있다.

* Process와 Thread의 차이 (Process/ Thread)

 실행되는 프로그램을 의미한다. / 프로그램의 부분을 의미한다.
가볍지 않다. / 가볍다.
종료하는 데에 시간이 많이 걸린다. / 종료하는 데에 더 적은 시간이 든다.
생성되는 데에 시간이 많이 걸린다. / 생성되는 데에 더 적은 시간이 든다.
Process들이 서로 소통하는 데에 시간이 많이 걸린다. / Thread들이 서로 소통하는 데에 더 적은 시간이 든다.
대부분 독립적이다. / Thread들은 메모리를 서로 공유한다.
Process들은 Data를 서로 공유하지 않는다. / Thread들은 Data를 서로 공유한다.
OS에 의해 서로가 따로 관리 받는다. / Thread들은 OS에 의해 단일 업무로 처리되기도 한다.
더 많은 자원이 든다. / 더 적은 자원이 든다.
